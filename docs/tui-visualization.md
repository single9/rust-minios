# TUI 視覺化介面操作指南

## 操作鍵總覽

| 按鍵 | 功能 |
|------|------|
| `F1` | 切換到儀表板 |
| `F2` | 切換到記憶體視圖 |
| `F3` | 切換到程序排程視圖 |
| `F4` | 切換到檔案系統瀏覽器 |
| `F5` | 切換到文字編輯器 |
| `F6` | 切換到 Shell |
| `q` | 離開（僅在儀表板生效）|
| `Ctrl+C` | 強制離開（任何模式）|

---

## F1：儀表板 (Dashboard)

總覽畫面，同時顯示各子系統的摘要：

```
┌──────────────────────────────────────────────────────────────┐
│  🖥  rust-minios v0.1.0  [F1] [F2] [F3] [F4] [F5] [F6]       │
├─────────────────────────┬────────────────────────────────────┤
│  開機訊息                │  系統統計                          │
│  [BOOT] 啟動中...        │  Tick: 1523                       │
│  [MEM]  256頁 x 4KB     │  程序數: 3                        │
│  [FS]   VFS 就緒        │  記憶體: 96KB / 1024KB used       │
│  [SCHED] 排程器初始化   │                                    │
│  [BOOT] 系統就緒        │                                    │
└─────────────────────────┴────────────────────────────────────┘
```

## F2：記憶體視圖 (Memory View)

視覺化 256 個記憶體分頁的使用狀態：

```
┌─ 記憶體管理員 ─────────────────────────────────────────────┐
│                                                            │
│     00 01 02 03 04 05 06 07 08 09 0A 0B 0C 0D 0E 0F       │
│  0: ██ ██ ██ ██ ██ ██ ██ ██ ██ ██ ██ ██ ██ ██ ██ ██      │
│  1: ██ ██ ▓▓ ▓▓ ▓▓ ▓▓ ▒▒ ▒▒ ░░ ░░ ░░ ░░ ░░ ░░ ░░ ░░      │
│  2: ░░ ░░ ░░ ░░ ░░ ░░ ░░ ░░ ░░ ░░ ░░ ░░ ░░ ░░ ░░ ░░      │
│  ...                                                       │
│                                                            │
│  ██ Kernel(藍)  ▓▓ Process(綠)  ▒▒ Reserved(紅)  ░░ Free  │
│                                                            │
│  統計: 總計 1024KB | Kernel 64KB | Process 32KB | Free 928KB│
└────────────────────────────────────────────────────────────┘
```

**顏色說明：**
- 🔵 藍色 `██` = Kernel 使用（前 16 頁，64KB）
- 🟢 綠色 `▓▓` = 程序使用（由各 PID 持有）
- 🔴 紅色 `▒▒` = Reserved（保留）
- ⬛ 灰色 `░░` = Free（可配置）

**理解要點：**
- 執行 `exec <name>` 後，觀察新的綠色方塊出現
- 執行 `kill <pid>` 後，對應的綠色方塊變回灰色
- 執行 `malloc <n>` 後，觀察綠色方塊增加

## F3：程序排程視圖 (Process View)

即時顯示程序狀態與排程器狀態：

```
┌─ 程序排程器 ──────────────────────────────────────────────┐
│                                                           │
│  PID  NAME      STATE      PRI  CPU TIME  MEMORY         │
│  ───  ────────  ─────────  ───  ────────  ──────         │
│    0  kernel    Running     10     1523    16KB           │  ← 綠色
│    1  init      Ready        8      340     8KB           │  ← 黃色
│    2  shell     Ready        5      218     8KB           │  ← 黃色
│    3  myapp     Blocked      5        8     16KB          │  ← 紅色
│                                                           │
│  就緒佇列 (Ready Queue):  [init(1)] → [shell(2)]          │
│  Blocked:                 [myapp(3)]                      │
│  Scheduler Tick:          1523                            │
│  Current:                 kernel(0)                       │
│                                                           │
└───────────────────────────────────────────────────────────┘
```

**觀察重點：**
- `cpu_time` 欄位隨時間遞增，顯示程序使用 CPU 的量
- 每 10 ticks 會發生一次 **搶佔（Preempt）**：Running → Ready → 佇列尾端
- 就緒佇列是 FIFO（先進先出），可觀察輪流順序

## F5：文字編輯器 (Editor)

類似 nano 的簡單文字編輯器，直接讀寫 VFS 檔案：

```
┌─ 編輯器：/home/notes.md ── [已修改] ──────────────────────┐
│                                                           │
│  1  # 我的筆記                                            │
│  2  ▌                                                    │  ← 游標位置
│  3  這是 rust-minios 的文字編輯器                         │
│  4                                                        │
│                                                           │
│                                                           │
│  Ctrl+S 儲存 | Ctrl+Q 離開 | Row:2 Col:1                 │
└───────────────────────────────────────────────────────────┘
```

**按鍵說明：**

| 按鍵 | 功能 |
|------|------|
| 任意字元 | 在游標位置插入 |
| `Enter` | 換行 |
| `Backspace` | 刪除游標前字元 |
| `Delete` | 刪除游標後字元 |
| `↑ ↓ ← →` | 移動游標 |
| `Ctrl+S` | 儲存至 VFS（執行 `Write` 系統呼叫）|
| `Ctrl+Q` | 離開編輯器，返回 Shell |

**開啟方式（Shell 中）：**
```bash
edit /home/notes.md     # 若檔案不存在會自動建立
```

## F6：Shell

命令列介面，模擬 Unix Shell：

```
┌─ Shell ───────────────────────────────────────────────────┐
│  rust-minios v0.1.0 - 輸入 'help' 查看指令               │
│                                                           │
│  $ help                                                   │
│  可用指令: ls cat mkdir touch rm ps kill exec             │
│            free malloc echo pwd cd tree edit help         │
│                                                           │
│  $ ps                                                     │
│  PID=0 NAME=kernel STATE=Running PRI=10 CPU=523           │
│  PID=1 NAME=init   STATE=Ready   PRI=8  CPU=120           │
│  PID=2 NAME=shell  STATE=Ready   PRI=5  CPU=89            │
│                                                           │
│  $ exec worker                                            │
│  [OK] Process 'worker' started (PID=3)                   │
│                                                           │
│  $ _                                                      │
└───────────────────────────────────────────────────────────┘
```

### 完整指令列表

```bash
# 程序管理
ps                   # 列出所有程序
exec <name>          # 啟動新程序
kill <pid>           # 終止程序

# 記憶體管理
free                 # 顯示記憶體統計
malloc <頁數>        # 手動配置記憶體頁
memfree <起始頁>     # 釋放記憶體

# 檔案系統
ls [路徑]            # 列出目錄（預設當前目錄）
cat <檔案>           # 顯示檔案內容
mkdir <目錄>         # 建立目錄
touch <檔案>         # 建立空檔案
rm <路徑>            # 刪除檔案或目錄
tree                 # 顯示完整目錄樹
edit <檔案>          # 開啟文字編輯器
pwd                  # 顯示當前目錄
cd <路徑>            # 切換目錄

# 其他
echo <文字>          # 輸出文字
help                 # 顯示說明
```

## 推薦的學習流程

1. **觀察開機**：啟動後在 F1 看開機訊息，了解各子系統初始化順序
2. **觀察記憶體**：F2 看初始記憶體佈局（Kernel 佔前 64KB）
3. **觀察程序**：F3 觀察 Round-robin 排程，注意 cpu_time 遞增
4. **操作程序**：F6 Shell 執行 `exec myapp`，再回 F3 看新程序出現
5. **操作記憶體**：Shell 執行 `malloc 10`，再回 F2 看分頁被佔用
6. **操作 FS**：Shell 執行 `touch /tmp/test.txt`，再到 F4 看檔案出現
7. **編輯檔案**：Shell 執行 `edit /home/readme.txt`，修改後 Ctrl+S 儲存
8. **驗證儲存**：Shell 執行 `cat /home/readme.txt`，確認修改生效
9. **終止程序**：Shell 執行 `kill <pid>`，觀察記憶體被釋放
